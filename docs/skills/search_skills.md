# 検索クローラー＆チューニングスキル仕様書（Index）

このファイルは、正確なリアルタイムデータ（天気、運行情報、ニュース等）を取得し、不正確な誤情報（SEOスパム、古い二次情報）を徹底排除して計算・検証するための「検索クロールスキル仕様書」です。

外部の検索（`searchWeb`）を実行する前に、必ず本仕様書を `readCodeFile` ツールで参照し、該当するカテゴリの検索パターン（スキル）に従って調査を組み立ててください。

## 各スキルの詳細とクロール戦略

### 1. 天気情報・気象データの取得 (weather)

- **適用対象**: 先生から特定の地域や日時の「天気」「降水確率」「気温」「傘が必要か」などを尋ねられた場合。
- **最優先事項 (重要)**: 天気情報を取得する際は、**必ず最優先で「気象庁 (JMA)」の公式サイト（jma.go.jp）から直接情報を取得してください**。民間や他の二次情報源よりも、気象庁の公式データを第一優先として参照します。
- **推奨検索クエリパターン (`searchWeb` 用)**:
  - 基本: `"気象庁 [地域名または都市名] 天気"` （例: `"気象庁 東京 明日の天気"`, `"気象庁 大阪 天気予報"`)
- **クローリング＆精査フロー (`fetchDynamicPage` 用)**:
  1. **【最優先】** 検索結果一覧の中から、必ず最初に『気象庁公式ページ（`https://tenki.jp/` 等）』のURLを見つけ、`fetchDynamicPage` を使って直接公式の気象データを開いて精査してください。
  2. ページ内のMarkdown化されたデータから、以下の「数値情報」を正確に抽出してください：
     - 天気（晴れ、曇り、雨など）
     - 降水確率（%）
     - 最高気温および最低気温（℃）
     - 雨が降る時間帯の目安
  4. 抽出したデータに基づいて、「傘が必要か」「どのような服装が適切か」を論理的かつ親身に判断し、伝えてください。

### 2. 電車の運行・遅延情報の取得 (train_status)

- **適用対象**: 特定の路線や地域の「電車の遅延」「運転見合わせ」「運行状況」について尋ねられた場合。
- **推奨検索クエリパターン (`searchWeb` 用)**:
  - 基本: `"Yahoo運行情報 [路線名]"` （例: `"Yahoo運行情報 山手線"`, `"Yahoo運行情報 中央線"`)
  - 補助: `"Yahoo運行情報 [地域名]"` （例: `"Yahoo運行情報 関東"`)
- **クローリング＆精査フロー (`fetchDynamicPage` 用)**:
  1. 検索結果の中から『Yahoo!路線情報 運行情報ページ（`transit.yahoo.co.jp/diainfo/` 等）』のURLを見つけ、`fetchDynamicPage` を使って開きます。
  2. 抽出されたMarkdown内から、以下の情報を完全に検証してください：
     - **現在の運行状態**: 「平常運転」か、「遅延」「運転見合わせ」「一部運休」が発生しているか。
     - **遅延の詳細**: 遅延理由（人身事故、大雨、信号トラブル等）や、影響が出ている区間。
  3. 平常運転の場合は先生を安心させ、遅延等の乱れが発生している場合は、迂回ルートの検討や移動時間に余裕を持つよう現実的なアドバイスを添えてください。

### 3. 最新ニュース・事実確認のファクトチェック (news_fact)

- **適用対象**: 最新のIT動向、製品情報、話題の事実、または真偽が不明な噂について尋ねられた場合。
- **推奨検索クエリパターン (`searchWeb` 用)**:
  - 基本: `"[キーワード] 最新情報 公式"` または `"[キーワード] ニュース 信頼性"`
- **クローリング＆精査フロー (`fetchDynamicPage` 用)**:
  1. 信頼性の高い一次ソース（公式サイト、Wikipedia、大手メディア等）のURLを見つけて開きます。
  2. 不確かな個人まとめブログや、煽り目的のまとめサイト（ゲハブログ等）の情報は完全に無視してください。
  3. 複数の異なる大手ニュースソース（例: ITmedia、Impress、PC Watch 等）の情報をクロスチェックし、共通して記述されている客観的な事実のみを論理的に再構成して先生に伝えてください。
