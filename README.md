# Yuuka - Discord Gemini Secretary Bot & Admin Dashboard

---

## ✨ 主な機能（できること一覧）

『Yuuka』は、Google の最先端 AI である **Gemini API** を活用した、Discord 秘書ボットと Web 管理者ダッシュボードのハイブリッドシステムです。

### 🤖 Discord 秘書 Bot 機能

*   **📐 早瀬ユウカによる親身なロールプレイ**
    *   ミレニアムサイエンススクールのセミナー会計「早瀬ユウカ」としての対話。
    *   あなたの困りごとに呆れつつも、論理的かつ親身になって解決策を提示し、実務的に優しくサポートしてくれます（小言や説教はほどほどに調整されています）。
*   **📋 タスク管理 (ToDo)**
    *   タスクの追加・一覧表示・完了・削除を自然な会話から行えます。
    *   期限日（`YYYY-MM-DD`）や優先度（低・中・高）も賢く認識して管理します。
*   **📅 予定管理 (スケジュール / Google カレンダー同期)**
    *   予定の登録・一覧表示・削除を会話を通じて実行可能。
    *   **Google カレンダーとの双方向同期**: 登録された予定は Google カレンダーに自動で同期されます。
    *   **ローカルタイマー・簡易リマインダー**: カレンダーを汚したくない「n分後に教えて」といったタイマーは、カレンダーに同期せず Discord 内で時間通りにメンション通知してくれます。
*   **💰 家計管理 (支出の記録・分析)**
    *   「〇〇に1200円使った」と話しかけるだけで、金額やカテゴリ（食費、日用品、娯楽など9種類）を自動判定して家計簿に記録。
    *   月間の支出サマリーやカテゴリ別支出内訳、直近履歴をチャットから即座に教えてくれます。
*   **📷 レシート解析 (OCR & 自動家計簿登録)**
    *   Discord にレシートの画像を貼り付けるだけで、Gemini が画像を解析。購入した商品を適切なカテゴリに自動分類し、家計簿へ自動的に一括登録してくれます。

### 🌐 Web 管理ダッシュボード

ブラウザからアクセスして、全てのデータを一元管理・視覚化できるプレミアムな管理ダッシュボードです。

*   **🔒 ユーザー管理とセキュアな認証システム**
    *   **管理者セットアップ**: 初回起動時、データベースにユーザーが存在しない場合は初期セットアップ画面が表示され、最初の登録者が自動的にシステム管理者になります。
    *   **一般ユーザー登録と招待コード**: 管理者が管理画面から発行した招待コードを使って、他のユーザーがアカウントを作成できます。
    *   **個別 API キー**: 各ユーザーは個人の Gemini API キーを登録して安全に使用できます。
    *   IPごとのログイン試行レート制限（5回失敗で15分間ロックアウト）や安全なセッションクッキー管理により、外部からの不正アクセスを防ぎます。
*   **🤖 複数 Bot 管理 (マルチ Bot 機能)**
    *   システム全体で共有するデフォルトBotの他、ユーザーが個別に独自の Discord Bot を作成・登録して動作させることができます。
    *   管理画面から Discord の名前やアバターをワンクリックで同期可能です。
*   **📋 タスク・予定・家計データのグラフィカルな管理**
    *   タスクの進捗、支出の推移グラフ、カテゴリ別の支出比率などをスタイリッシュに視覚化します。
    *   手動での追加・削除に加え、Web 画面からの AI レシートスキャナー画像アップロードもサポートしています。
*   **📝 Playbook 自動化（手順書機能）**
    *   Bot に実行させたい任意の操作手順を Playbook（手順書）として作成・管理できます。
    *   作成した Playbook は Cron式を用いて実行スケジュール（例: 「毎朝9時」「毎週月曜」など）を設定でき、定期的な自動実行と実行履歴の追跡が可能です。
*   **🌐 Google カレンダー連携の動的設定**
    *   各 Bot ごとに Google OAuth 連携を行い、同期対象のカレンダーを管理画面から動的に切り替え・設定できます。

## 🚀 セットアップ手順

### 1. ユーザー / Bot 依存関係のインストール

プロジェクトディレクトリに移動し、pnpm を使用してパッケージをインストールします。

```bash
pnpm install
```

### 2. 設定ファイルの作成

テンプレートファイル `example.yaml` をコピーして、`config.yaml` を作成します。

```bash
cp example.yaml config.yaml
```

`config.yaml` を開き、必要な設定項目を記述します：
*   **`DB_PATH`**: SQLite データベースファイルの保存パス（デフォルト: `./data/yuuka.db`）。
*   **`REDIS_URL`**: インメモリデータキャッシュ用の Redis 接続 URL。
*   **`PORT` / `HOST`**: 管理画面サーバーがリスンするポートとホスト名設定。
*   **`GOOGLE_CLIENT_ID` / `GOOGLE_CLIENT_SECRET`**: Google OAuth 認証情報。
*   **`BASE_URL`**: 外部からアクセス可能な HTTPS ベース URL。
*   **`YUUKA_ENCRYPTION_SECRET`**: データベースに保存する Discord トークンや API キーの暗号化に使用するシークレットキー。

## 🏃 起動と開発

### 開発モード (ホットリロード有効)
コードの変更を監視し、自動でサーバーが再起動します。

```bash
pnpm dev
```

### プロダクションビルド & 起動
TypeScript のコンパイルを行い、本番環境用として最適化されたビルドを起動します。

```bash
# コンパイル (dist/ 以下に出力されます)
pnpm build

# 本番サーバーの起動
pnpm start
```

起動後、ブラウザで `http://localhost:7854` (または `config.yaml` で設定したポート/ホスト) にアクセスすると、管理者用ダッシュボードが開きます。

1.  **初期セットアップ**: 最初のユーザーとして管理者アカウントを登録します。
2.  **デフォルトBotのトークン設定**: 管理者アカウント作成後、システム全体のデフォルトBotトークンを入力して起動します。
3.  **使い方の確認**: ログイン前であっても、ログイン画面下部の「使い方ガイドを読む（`/usage`）」から基本的な使用方法やコマンド例をいつでも確認できます。

## ⚙️ systemd による常時稼働 (Linux環境)

同梱されている `yuuka.service` テンプレートを利用して、Linuxサーバー上でサービスとしてデーモン化できます。

1.  `yuuka.service` ファイルをお使いの環境のパス (例: `WorkingDirectory` や `ExecStart` のNode.jsパスなど) に合わせて編集します。
2.  サービスファイルを配置します：
    ```bash
    sudo cp yuuka.service /etc/systemd/system/yuuka.service
    ```
3.  デーモンをリロードしてサービスを有効化・起動します：
    ```bash
    sudo systemctl daemon-reload
    sudo systemctl enable yuuka.service
    sudo systemctl start yuuka.service
    ```
4.  ステータスの確認：
    ```bash
    sudo systemctl status yuuka.service
    ```
